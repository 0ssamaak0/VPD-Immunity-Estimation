# VPD Immunity & Outbreak Dashboard

Interactive dashboard for the VPD-Immunity-Estimation project.
Runs entirely offline — no server required.

## Open the dashboard

Double-click `dashboard/index.html` (or open it in any modern browser via
`file://…`). No internet connection needed; Plotly is vendored locally.

## Regenerate data

If you re-run the R pipeline and want to update the baked data layer:

```bash
conda run -n mmed Rscript dashboard/build_data.R
```

This overwrites `dashboard/data.js` with fresh posteriors and simulation seeds.
Refresh `index.html` afterwards.

## Deploy to GitHub Pages

1. Push the entire `dashboard/` folder to the `main` branch (or a `gh-pages`
   branch) of your GitHub repository.
2. In **Settings → Pages**, set the source to the branch and `/dashboard` as the
   folder (or `/` if you publish from `gh-pages`).
3. GitHub Pages will serve `index.html` as a static site — no build step needed.

## Files

| File | Role |
|------|------|
| `index.html` | Page shell, tab layout, script loading order |
| `style.css` | Report palette & responsive layout |
| `app.js` | All Plotly charts + interactive controls |
| `data.js` | Baked data globals (`window.DASHBOARD_DATA`) — generated by `build_data.R` |
| `reedfrost.js` | Client-side Reed-Frost SEIR engine (`window.ReedFrost`) |
| `plotly.min.js` | Vendored Plotly v2.35.2 — no CDN dependency |
| `build_data.R` | R script that generates `data.js` from pipeline outputs |
